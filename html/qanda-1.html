What is DevOps?

  • A tool or set of tools to increase automation
  • A team of senior operations engineers that enable infrastructure operation
  • A team of developers that own their application in production
  • Any efficiencies gained in the feedback loop to speed up the delivery of software to customers

What are two benefits of a continuous delivery pipeline?

  • Software is always ready to release and there is one team working on a single pipeline
  • Software is always ready to release and developers no longer need to interact with operations staff
  • Multiple versions of a software application are run in production in parallel, and developers no longer need to interact with operations staff
  • Multiple versions of a software application are run in production in parallel, and errors in the pipeline can be ignored, allowing developers to continue working on important new features

What role does CodePipeline play in a continuous delivery pipeline?

  • Defines a workflow to orchestrate the build, testing, and deployment of your code every time it changes
  • Synchronizes code between two systems, for example, GitHub and S3
  • Automatically merges code branches based on predefined workflows
  • Performs static analysis of source code to check for known vulnerabilities
