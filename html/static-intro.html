Static Website: Learning Objectives

Learning Objectives: Script

In this lesson, we're going to use the website I'm using to walk through this course to demonstrate how to use Continuous Delivery and AWS tools to deploy changes to production based on recent code commits. Other lessons in this course will build on this example and add additional use cases and complexity to this solution. So, this is going to be rather "meta" as I'm going to walk through the course website in this video. You'll see how to make code changes that go through a deployment pipeline composed of stages and actions in CodePipeline to update the course website itself. All of the code to do this will be provided to you and regularly updated.

To start things off, you will click on a Launch Stack button to launch a stack based on a CloudFormation template. This CloudFormation template is committed as code in my GitHub repository.

Next, we will go through the resources in the CloudFormation template that automates the provisioning of several resources including an AWS CodePipeline pipeline, an AWS CodeBuild project, some AWS IAM Roles, and an AWS S3 Bucket. CodePipeline is also configured to integrate with various providers such as GitHub and CodeBuild.

By the end of this lesson you will be able to -

2.1 Configure course prerequisites

2.2 Manually provision a static website

2.3 Describe how to orchestrate software workflows in CodePipeline

2.4 Store software assets in CodeCommit

2.5 Build and test software in CodeBuild

2.6 Describe the architecture and launch the solution

2.7 Provision AWS resources in CloudFormation

2.8 Orchestrate the solution in CodePipeline
