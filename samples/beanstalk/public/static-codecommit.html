Introduction

"AWS CodeCommit is a fully-managed source control service that makes it easy for companies to host secure and highly scalable private Git repositories. CodeCommit eliminates the need to operate your own source control system or worry about scaling its infrastructure. You can use CodeCommit to securely store anything from source code to binaries, and it works seamlessly with your existing Git tools." (Source)

In this sublesson, you will create a new CodeCommit repository using the AWS Console.

Create a CodeCommit Repository and configure Git credentials

Follow the steps below to configure Git credentials so that you can connect to the CodeCommit repository that you will create later this sublesson -

  • Go to the AWS CodeCommit Console
  • Select US East (N. Virginia) from the region menu
  • Select Create repository
  • Enter a unique Repository name and provide a description and Click Create repository
  • A window named Connect to your repository will display. Ensure that the HTTPS radio button is selected for the Connection type. Under the Steps to clone your repository section, click on the IAM user link.
  • Go to the Permissions tab and add ensure the following policies have been assigned to your IAM user by selecting Add permissions and going through the steps to select the policies listed below -
    • AWSCodeCommitPowerUser (or an appropriate CodeCommit managed policy)
    • IAMSelfManageServiceSpecificCredentials
    • IAMReadOnlyAccess
  • Select the Security credentials tab and choose the Generate from the HTTPS Git credentials for AWS CodeCommit section. Choose Download credentials and save the file on your computer for future reference.
  • Go back to the AWS CodeCommit Console and copy the contents of the Git command and paste into your Terminal window
  • When prompted, enter the username and password from the credentials file you downloaded. You should have successfully cloned your newly-create CodeCommit Git repository.

See Introducing Git Credentials for more information.
