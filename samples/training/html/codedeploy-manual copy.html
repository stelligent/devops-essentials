Static Website: Manually provision static website

Introduction

"Amazon Simple Storage Service (Amazon S3) is object storage with a simple web service interface to store and retrieve any amount of data from anywhere on the web." (Source)

In this sublesson, you will upload HTML and other files to an Amazon Simple Storage Service (S3) bucket and configure it as a static website. Later in this lesson, you will automate the process you're performing manually.

For more information on using S3 for static website hosting, see How Do I Configure an S3 Bucket for Static Website Hosting?

Create and configure an S3 Bucket for Static Website Hosting

Follow these steps to create and configure an S3 bucket for static website hosting -

  • Go to the Amazon S3 console and select Create Bucket
  • Enter a globally-unique Bucket name. See Bucket Restrictions and Limitations for bucket naming constraints and other limits.
  • Select the US East region
  • Select Next
  • From the Set Properties panel, select Next
  • From the Set Permissions panel, select Next
  • From the Review panel, select Create bucket
  • Click the newly-created bucket and go to the Properties tab and select Static website hosting
  • Choose the Use this bucket to host a website radio button and type index.html for Index document and select Save
  • Select the Objects tab. Select Upload, then drag and drop the contents of the html directory from your cloned GitHub repository.
  • Choose Next
  • Choose Manage public permissions and select the Read checkbox for Objects and Everyone
  • Choose Next
  • Choose Next
  • Choose Upload
  • Select the Properties tab
  • Select the Static website hosting and click on the Endpoint URL. The static website should correctly display.
